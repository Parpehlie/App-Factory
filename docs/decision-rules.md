# Portfolio decision rules

## Apps with engagement-gated monetization

When the paid gate is reached only after repeated use, the review window must follow the
product's actual conversion moment. IronHale gates session 7, which lands around day 14
at three sessions per week and day 21 at two sessions per week.

| Review | Metrics | Allowed verdict |
| --- | --- | --- |
| Day 14 | Organic downloads, D1, D7, install → workout 1, install → workout 3 | Kill only when acquisition is clearly dead |
| Day 28 | Install → workout 6, session-7 paywall → purchase, install → paid | Kill, maintain, or double down |

| Metric | Kill | Double down |
| --- | ---: | ---: |
| Organic downloads (14 days) | < 50 | ≥ 300 |
| D1 | < 20% | ≥ 25% |
| D7 | < 8% | ≥ 12% |
| Install → workout 1 | < 20% | ≥ 35% |
| Install → workout 3 | — | ≥ 18% |
| Install → workout 6 | — | ≥ 10% |
| Session-7 paywall → purchase | < 8% | ≥ 20% |
| Install → paid (day 28) | 0% | ≥ 2.0% |

Verdicts are made only during the weekly review, never from an incomplete cohort.
